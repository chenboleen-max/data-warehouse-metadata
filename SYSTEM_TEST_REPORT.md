## 测试步骤

### 1. 部署系统

```bash
# 克隆代码
git clone <repository-url>
cd kiro_web

# 配置环境变量
cp .env.example .env
vi .env  # 修改密码和密钥

# 启动服务
docker-compose up -d

# 等待所有服务启动
sleep 60

# 检查服务状态
docker-compose ps
```

**预期结果**: 所有服务状态为 `Up (healthy)`

### 2. 验证服务可用性

```bash
# 检查前端
curl -f http://localhost/ || echo "前端服务异常"

# 检查后端健康状态
curl -f http://localhost:8080/actuator/health || echo "后端服务异常"

# 检查 MySQL
docker-compose exec mysql mysql -u root -p<password> -e "SELECT 1" || echo "MySQL 异常"

# 检查 Redis
docker-compose exec redis redis-cli ping || echo "Redis 异常"

# 检查 Elasticsearch
curl -f http://localhost:9200/_cluster/health || echo "Elasticsearch 异常"
```

**预期结果**: 所有服务响应正常
